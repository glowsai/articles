# 安いクラウドGPU：最低時給は最低コストではない

GPUの表示時給には、モデル取得、ストレージ、ネットワーク、待機、失敗時の再実行は通常含まれません。安価なホストが何度も失敗したり、セットアップが長い場合、少し高いが安定したセッションの方が早く、結果的に安くなります。

価格と課金規則は変わるため、実際の見積もりでは各候補の一次情報を使用してください。たとえば[RunPodの料金](https://www.runpod.io/pricing)と[Glows.ai](https://glows.ai/)は、GPU、ストレージ、課金条件を確認する出発点になります。

`実コスト = GPU計算 + ストレージ + 転送 + 設定・待機時間 + 再実行`

## 3つの場面

**45分のComfyUI実験**では、起動とモデル取得の時間が時給差より重要になり得ます。**20時間の微調整**では、checkpointと永続ディスクが決定的です。**72時間以上の学習**では、複数GPUのネットワーク、容量、信頼性が最優先になります。

## 比較の手順

同じイメージ、モデル、データ、タスクで候補を試します。利用可能になるまでの時間、処理時間、停止後の保存費用、復元の可否、実請求を記録してください。GPU型番とVRAMが違う価格は比較対象にしません。

中断可能でバックアップ済みのバッチ作業なら低価格を追えます。締切があるデモ、反復的な制作、重要な学習では、運用の確実性に価値があります。
